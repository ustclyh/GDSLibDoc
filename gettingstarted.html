Get Started

程序包包含 ElementStructures, Devices以及ChipConstructors 三个层级.

ElementStructures 封装了 gdstk 的三种对象, 即 gdstk.RobustPath, gdstk.Polygons 和 gdstk.Reference. ElementStructures中的类 Paths, Polygons 和 References 均包含了存储对应 gdstk 对象的列表. 此外 Paths 可单独使用, 用于生成简单结构, 方便基于 FDTD 的仿真.

Devices 提供了构造器件的方法, 它是 gdstk.RobustPath, gdstk.Polygons 和 gdstk.Reference 的子类, 继承了一切属性和方法. Devices 提供的方法 generate_cell 将自动融合其三个父类提供的列表, 将全部对象放置于一个 PCell 内, 并将此 PCell 注册于一可以随时调用的字典, 方便后续的芯片级操作.

ChipConstructors 提供了多种生成 Devices 引用的方法, 完成芯片的最终 gds 导出.

下面将按照上述层级对程序包的使用进行介绍.

ElementStructures 层级

Paths 类

Waveguide 类调用 gdstk.RobustPath 生成波导几何结构. 我们结合例子说明此类的基本使用方法.

import numpy as np
import gdstk
from ElementStructures import Paths

lib = gdstk.Library('library')

path = Paths()

# plot a racetrack ring
path.new_path(start=[0, 5], width=2, layers=0)
path.add_segment(length=30, direction=0)
path.add_turn(radius=10, angle=np.pi)
path.add_segment(length=30)
path.add_turn(radius=10, angle=np.pi)
# add a copy of current path, otherwise the follow mirror operation will influence this added path
# change the 'add_copy' parameter to 'false' to see the different
path.end_current_path(add_copy=True)

# mirror the above ring and add it to the path list
path.curPath.mirror([1, 0])
path.end_current_path()

# plot another waveguide, which is a straight one
path.new_path(start=[-35, 0], width=1, layers=1)
path.add_segment(length=100, direction=0)
path.end_current_path()

# generate PCell and add it to the GDS library
lib.add(path.generate_pcell('MAIN'))
# write GDS file
lib.write_gds('example.gds')

path = Paths() 用于类的初始化, 随后我们便可以反复调用 Paths 提供的方法生成结构. 每当需要生成一条新路径, 调用 new_path 方法开始路径, 随后便可反复调用 add_* 方法加入不同结构, 当完成一条路径绘制后, 调用 end_current_path 方法结束绘制. 可以通过以上过程多次添加不同路径. 每一对 new_path 和 end_current_path 都会为 Paths.path_list 添加一个新的元素. 当完成全部路径绘制后, 调用 generate_pcell 方法, 便可将列表中已添加的全部路径对象放置在一个 gdstk.Cell 中, 此方法将放置路径后的 gdstk.Cell 对象作为返回值返回.

Note

对于熟悉 gdstk 包的用户, Paths 还可以以一种更灵活的方式使用. end_current_path 方法的参数 add_copy 若为真, 则加入路径列表的元素为当前操作路径对象的拷贝. 此时可以直接操纵 Paths 类的属性 curPath, 使用 gdstk.RobustPath 提供的方法, 对其进行镜像, 旋转等操作, 随后再次调用 end_current_path 便能够将变换后的路径再次加入 Paths.path_list.

Polygons 和 References 类

这两个类用于构建 Devices 类, 不建议用户单独调用, 不提供操作说明. 如果有需要, 可以自行阅读代码 (代码量很小).

Devices 层级

Device 类

Device 为定义任意器件提供了通用的方法和参数. Device 类为 Paths, Polygons 和 References 的子类

基于 Device 类的器件构造方法

我们结合例子说明器件的定义方法和注意事项.

from Devices import Device
from Devices import LBend
import numpy as np

class RaceTrackSpiralOneSide(Device):
    SBendRa = 100.0 / 2

    def __init__(self, name):
        super().__init__(name)
        # All the pattern concerning parameters should be masked.
        self.width = 4.75  # waveguide width
        self.gap = 3  # waveguide width
        self.innerL = 1500
        self.N = 10
        self.total_length = 0
        self.ports_name_map = {'i': 0, 'o': 1}

    ##############################################
    # draw device method, also save the ports
    def draw(self):
        Ra = RaceTrackSpiralOneSide.SBendRa

        # generate L bend PCell
        sbend = LBend("LBend50")
        sbend.width = self.width
        sbend.dx = Ra
        sbend.dy = Ra
        # sbend.draw()
        sbend.generate_cell('SBend50')

        # add the S bend PCell references
        dd = self.gap + self.width
        for i in range(0, 2*self.N):
            self.add_reference(Device.devList['SBend50'], origin=[self.innerL + dd * i, -Ra - Ra / 2 - dd * i])
            self.add_reference(Device.devList['SBend50'], origin=[-self.innerL - dd * i, Ra + Ra / 2 + dd * i],
                            rotation=np.pi)
            self.add_reference(Device.devList['SBend50'], origin=[-self.innerL - dd * i - Ra, -Ra / 2 - dd * i - dd],
                            rotation=3 * np.pi / 2)
            self.add_reference(Device.devList['SBend50'], origin=[Ra + self.innerL + dd * i, Ra / 2 + dd + dd * i],
                            rotation=np.pi / 2)
            self.total_length += sbend.total_length * 4

        i = 2 * self.N
        self.add_reference(Device.devList['SBend50'], origin=[self.innerL + dd * i, -Ra - Ra / 2 - dd * i])
        self.total_length += sbend.total_length

        # connect the clockwise and anti-clockwise spirals
        # the euler bend shape is unchanged
        # when the racetrack being wider, add longer straight segments
        self.new_path([-200, 3*Ra/2], width=self.width)
        self.total_length += self.add_spline([2*200, -3*Ra, 0, 0])
        self.end_current_path()

        self.new_path([-self.innerL, 3 * Ra / 2], width=self.width)
        self.add_segment(self.innerL - 200)
        self.end_current_path()
        self.total_length += self.innerL - 200

        self.new_path([self.innerL, -3 * Ra / 2], width=self.width)
        self.add_segment(self.innerL - 200, direction=np.pi)
        self.end_current_path()
        self.total_length += self.innerL - 200

        # add straight waveguides
        for i in range(0, 2*self.N):
            self.new_path([-self.innerL - Ra-dd*i, Ra/2+dd*i], width=self.width)
            self.add_segment(Ra + 2 * dd * i + dd, direction=3 * np.pi / 2)
            self.end_current_path()
            self.total_length += Ra + 2 * dd * i + dd

        for i in range(0, 2 * self.N):
            self.new_path([self.innerL + Ra + dd * i, Ra / 2 + dd * i + dd], width=self.width)
            self.add_segment(Ra + 2 * dd * i + dd, direction=3 * np.pi / 2)
            self.end_current_path()
            self.total_length += Ra + 2 * dd * i + dd

        for i in range(0, 2*self.N):
            self.new_path([-self.innerL - dd * i, -3 * Ra / 2 - dd - dd * i], width=self.width)
            self.add_segment(self.innerL * 2 + dd * i * 2 + dd, direction=0)
            self.end_current_path()
            self.total_length += self.innerL * 2 + dd * i * 2 + dd

        for i in range(0, 2 * self.N):
            self.new_path([-self.innerL - dd * i - dd, 3 * Ra / 2 + dd + dd * i], width=self.width)
            self.add_segment(self.innerL * 2 + dd * i * 2 + dd, direction=0)
            self.end_current_path()
            self.total_length += self.innerL * 2 + dd * i * 2 + dd

        i = 2 * self.N - 1
        self.new_path([-self.innerL - dd * i - dd, 3 * Ra / 2 + dd + dd * i], width=self.width)
        self.add_segment(Ra - dd + self.width / 2, direction=-np.pi)
        self.end_current_path()
        self.total_length += Ra - dd + self.width / 2
        self.ports.append(self.curPath.spine()[-1, :])

        self.new_path([self.innerL + Ra + dd * i + dd, - Ra / 2 - i * dd - dd], width=self.width)
        self.add_segment(Ra + 2 * (i + 1) * dd, direction=np.pi / 2)
        self.end_current_path()
        self.total_length += Ra + 2 * (i + 1) * dd

        curPosition = self.curPath.spine()[-1, :]
        curPosition += np.array([50, 50])
        self.add_reference(Device.devList['SBend50'], origin=curPosition,
                        rotation=np.pi)
        self.total_length += sbend.total_length
        self.ports.append(curPosition)


if __name__ == "__main__":
    spiral = RaceTrackSpiralOneSide('Test')
    # spiral.draw()
    spiral.generate_cell()
    print(spiral.total_length/1000)
    print(spiral.ports)
    Device.write_gds(Device.devList['Test'], 'debug.gds')

一个 Device 类的定义非常简单, 它只需集成 Device 类即可. 用户需要根据器件需求, 为此器件添加需要的属性. 并给出绘制此器件的 draw() 方法, 绘制过程可以调用 Paths, Polygons 和 References 所提供的方法.

Device 类有两个非常重要的属性, ports_name_map 和 ports, 前者为字典, 后者为 N*2 矩阵. ports_name_map 储存具备可读性的端口名与端口序号之间的对应关系, ports 按照端口序号存储相应端口的二维坐标. ports_name_map 在类初始化时便建议定义, 并不要轻易变动; ports 可在方法 draw() 中随着几何定义的给出逐步确定. CircuitConstructor 将基于以上属性, 完成基于端口的线路连接.

Device 的 generate_cell(name) 方法将自动调用 draw() 方法, 并将全部元素放置于 Cell 生成 Pcell. 该 PCell 对应当前几何参数对应的几何结构, 将此 PCell 以字典存储于 Device 的静态属性 devList 中, 并将端口信息同步存储于静态字典 devDict. 在使用 Device 子类定义的器件时, 请为不同参数的器件定义不同的名称, 否则后定义的器件将覆盖先前定义的器件.
