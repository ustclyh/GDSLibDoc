Paths

class ElementStructures.Paths(precision=0.001)

Bases: object

Paths 类

该类用于构建由多边形定义的路径。其方法分为两大类：路径初始化和结束；路径的绘制。

基本使用逻辑为：使用 new_path 方法初始化一条路径，随后通过若干 add_* 格式的方法，加入 直线，弯曲以及spline 等路径。当一条路径绘制完成后，调用 end_current_path 方法，将此路径添加入 path_list 。 重复以上过程，将若干路径加入 path_list ，当完成全部路径定义后，调用 generate_pcell 方法， 从当前 path_list 生成Pcell。

当连续调用 add_* 格式的方法时，该类会自动记录路径末端的坐标、斜率以及曲率，用于所添加路径的自动平滑拼接。

Variables:

  • poth_list (list of gdstk.RobustPath object) – 存储路径对象的表.

  • curPath (gdstk.RobustPath) – 当前正在操纵的路径对象

  • curPosition (array[1][2]) – 当前路径末端坐标

  • curWidth (double) – 当前路径末端宽度

  • curAngle (double) – 当前路径末端角度

  • curCurvature (double) – 当前路径末端曲率

Parameters:

precision (double) – 求解精度

end_current_path(start=None, width=None, layers=0, max_evals=50000, add_copy=False, auto_new=False)

将当前路径添加入path_list，并结束当前路径，并根据情况开始一段新路径. 当start参数非空时，会开始一段新路径.

Parameters:

  • start (array[1][2]) – 新路径起始坐标

  • width (array) – 路径宽度

  • layers (array) – 放置路径的layer

  • max_evals (int) – 生成多边形时，计算的点数. 当需求的多边形顶点超过此数后，路径将生成不全.

  • add_copy (boolean) – 如果为真，加入path_list的是当前路径的一个拷贝，接下来如果进一步对此路径进行镜像等操作，将不会影响已经添加路径的结果.

  • auto_new (boolean) – 当start为空时，若auto_new为真，将自动在原地，以原先宽度初始化一段新路径

Device

class Devices.Device(name)

Bases: Paths, Polygons

Device 类

该类应当是 Devices 中所有器件的父类， Device 提供了器件构建的通用方法。 此外，在一次芯片构建过程中，所需所有的PCell的实例，均需以字典的形式存储于类的静态属性 devList 中，相应的 端口信息，以字典的形式存储于静态属性 devDict 中。

此类继承了 Paths 和 Polygons，因此其中包含存储器件所包含路径以及多边形的列表。此外，此类还添加了 与端口相关的属性，用于系统级的芯片构建。

当器件几何结构定义完成后，即可调用 generate_cell 方法，基于当前几何结构生成 PCell ，并将此 PCell 注册。 generate_cell 方法将自动合并 path_list 和 polygon_list。

该类还提供了静态方法，用于将由程序包 gdsfactory 生成的 Component 对象直接注册至器件的 PCell 列表， 用于此程序包与基于 gdsfactory 所定义器件库的兼容性匹配。

Variables:

name (str) – 器件的名称.
